# Sät | A platform for restaurants to sell remaining meals

## Overview

The project was built using Nest.JS, Meilisearch, Redis and Supabase.

There are two types of accounts, clients and restaurants. Both can add basic details and restaurants can upload their photo and set their location. Then the address is retrieved using the coordinates.

## Catalogue

Clients can search for restaurants and offers using the catalogue. The catalogue has searching, sorting and filtering functionality. The search is done using Meilisearch. Data is cached using Redis, if necessary. There are different types of filter modes (nearby for example).

## Restaurant

Restaurant managers can add menu items and offers. They can also update the data about them and the restaurant itself. When an offer is accepted by a client, it is considered activated and an order is created. When the order is paid and completed, the client must use the code generated by the app to finalize the order completion.
